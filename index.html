Architecture decisions
monorepo — logic written once
packages/core, packages/api, packages/i18n shared between mobile and web. Change business logic once, applies everywhere.
prompt caching in conversation
Grit's system prompt is long. Without caching, every turn pays the full prompt. With caching, only new tokens — up to 90% cost reduction.
batch vs. realtime by urgency
Chat → immediate streaming. Weekly reviews → Batch API at 3AM (50% discount, user sees result in the morning).

"Gritly forced me to think about LLM cost architecture as a design constraint — you have to make the product economically viable before it scales."

Architecture

Core patterns

The principles I apply across every system — with real production examples.

// 01
Modular architecture
Independent stages that can be updated, scaled, or replaced without touching the rest. Critical in GTM where ICP, models, and data sources change constantly.
LeonLeads: 6 modules.
Instantly breaks → swap send module only.
// 02
Event-driven vs. scheduled
Event-driven when reaction speed impacts the outcome. Scheduled when timing is predictable and batch makes economic sense.
Lead Scoring: webhook, fires in seconds.
Pipeline Monitor: every Monday.
// 03
Waterfall enrichment
Cheapest data source first. Only call the next if needed. One of the highest-impact cost decisions in LLM systems.
CSV → web search → Claude API (synthesis only).
ICP filter spends zero tokens on non-fit accounts.
// 04
Cache as first-class citizen
Design decision, not afterthought. DB cache, prompt caching, TTL — each used where it fits best.
Lighthouse: Smart Review = zero tokens.
Gritly: prompt caching = 90% cost reduction.
// 05
Human-in-the-loop
Not a limitation — a deliberate design for actions where the cost of error is asymmetric and irreversible.
LeonLeads: AI generates 5 emails,
rep approves in 3 min. Nothing goes out without sign-off.
// 06
LLM cost architecture
Model selection as an architectural constraint from day one. The right model per task, not the best model for everything.
Haiku: high-freq · Sonnet: high-quality
Batch API: non-urgent (50% off) · No LLM: when Supabase is enough.
